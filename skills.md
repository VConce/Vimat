# Habilidades y reglas de trabajo

- Rigor matemático: revisar definiciones, hipótesis, cuantificadores, casos límite y consistencia de notación antes de proponer cambios.
- Nivel oposición: los temas deben ser sólidos para tribunal, pero explicables para opositores con base en física, arquitectura, ingeniería u otras formaciones no puramente matemáticas.
- Diagnóstico útil: cada sugerencia debe indicar documento, página, zona aproximada, qué cambiar, por qué conviene cambiarlo y una propuesta concreta de redacción o fórmula.
- Fórmulas: las páginas web deben renderizar LaTeX o equivalente mediante MathJax/KaTeX. Preferir notación estándar: \(f^{(k)}(a)\), \(D^k f\), \(A\subset\mathbb R^n\), etc.
- Word/PDF: revisar artefactos de exportación, en especial cuadros de texto fantasma, matrices rotas, glifos cortados, flechas/vectoriales mal renderizadas y paréntesis de derivadas sucesivas.
- Web de revisión: mantener menús contraíbles/expandibles, filtros, notas, marcado de revisado e importación/exportación JSON suficiente para restaurar estado y sugerencias.
- Separación editorial: distinguir notas internas del editor de contenido final para opositores; no dejar frases tipo “decide tú” o instrucciones de redacción en versiones limpias.
- Didáctica LOMLOE: cuando se trabaje con PD/UD, conectar Situaciones de Aprendizaje, saberes básicos, competencias específicas, criterios de evaluación y Perfil de Salida.
- Memoria activa: si el usuario corrige una decisión matemática, normativa o de estilo, actualizar `memory.md` para no repetir el error.
